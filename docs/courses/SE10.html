About this course

Human-Machine Interaction (HMI) bridges cognitive science, design, and AI to produce interfaces that are effective, efficient, and inclusive. Students move from foundational perception and cognition theory through structured UX design processes, multimodal input channels, and accessibility standards, concluding with AI-augmented interaction patterns such as adaptive UIs and conversational agents. Every concept is anchored in a semester-long design-and-evaluation project.

Course format. Thirteen weeks, four contact hours each: a two-hour lecture (concepts and theory) and a two-hour practice session. The course is project-based; teams carry one running project end to end and present it three times, in weeks 5, 8, and 13.
What you will build

Designed, prototyped, and evaluated a human-machine interface for a real system: conducted user research and competitive analysis, built low- and high-fidelity Figma prototypes, ran moderated usability tests with think-aloud protocols and SUS scoring, audited for WCAG 2.2 compliance, added a voice or gesture modality, and integrated an AI-powered adaptive or conversational layer, all documented in a design system and a final usability report with measurable improvement evidence.

Expected outcomes

  • Apply cognitive models including mental models, affordances, Fitts' Law, and information-processing theory to predict and analyze interaction behavior
  • Conduct structured user research through contextual inquiry, interviews, surveys, and task analysis
  • Design and iterate UX artifacts from personas and journey maps through low-fidelity wireframes to interactive Figma prototypes
  • Plan and execute moderated usability tests, analyze results with SUS and task-success metrics, and drive design revisions from evidence
  • Incorporate multimodal input channels including voice, gesture, and gaze into an interface design with appropriate feedback models
  • Audit an interface for WCAG 2.2 accessibility compliance and remediate identified barriers
  • Design and evaluate AI-augmented interaction patterns: adaptive interfaces, conversational UI, and context-aware agents
  • Apply A/B testing and analytics instrumentation to measure the impact of design changes in production
  • Deliver a complete design system with component library, interaction specifications, and a usability evaluation report

Theoretical foundations

The concepts and results this course rests on.

  • Human information processing: perception, attention, and working memory constraints
  • Mental models and the gulf of evaluation and execution (Norman)
  • Affordances, signifiers, mappings, and feedback in interface design
  • Fitts' Law and motor-control models for pointing and target acquisition
  • The GOMS family of models for predicting user task times
  • Design thinking as an iterative problem-solving framework
  • Gestalt principles applied to visual interface layout
  • Multimodal fusion theory: combining voice, gesture, and gaze signals
  • Speech recognition architectures and natural language understanding pipelines
  • WCAG 2.2 principles: perceivable, operable, understandable, robust
  • Adaptive and personalized systems: user modeling and context-aware recommendation
  • Conversational UI and dialogue management design patterns
  • A/B testing theory: statistical power, Type I/II error, and sequential testing

Weekly schedule 13 weeks · lecture + practice

Foundations
Wk 1
Cognitive Foundations of HCI
LectureIntroduce the human information-processing model: perception, attention, working memory, and long-term memory constraints on interface design.
PracticeAnalyze an existing interface for violations of cognitive load and perception principles.
ProjectProject domain and target user group are chosen; initial domain research is begun.
Wk 2
Mental Models, Affordances, and Fitts' Law
LectureCover Norman's action cycle, mental models, affordances, signifiers, and Fitts' Law as a quantitative model of motor interaction.
PracticeMeasure Fitts' Law parameters in a controlled pointing experiment and analyze interface layout against the model.
ProjectCognitive walkthrough of the chosen domain's current interface is produced.
Wk 3
User Research Methods
LecturePresent contextual inquiry, semi-structured interviews, surveys, task analysis, and affinity diagramming.
PracticePlan and conduct a mini contextual inquiry session; synthesize findings into an affinity diagram.
ProjectUser research plan is written; at least three user interviews are conducted and synthesized.
Design
Wk 4
UX Design Principles and Design Thinking
LectureIntroduce design thinking (empathize, define, ideate, prototype, test), Gestalt visual principles, and the eight golden rules of interface design.
PracticeRun an ideation workshop: produce personas, user journey maps, and a set of competing low-fidelity sketch concepts.
ProjectPersonas, journey maps, and three competing low-fidelity concepts are produced.
Wk 5
Prototyping and Specification MilestonePresentation
LectureDiscuss fidelity levels in prototyping, interaction specification techniques, and design system foundations: type scales, color roles, spacing tokens.
PracticeTeams present their project specification: user research findings, chosen concept, Figma wireframes, and proposed evaluation plan.
ProjectApproved specification with low-fidelity wireframes, research synthesis, and evaluation plan is delivered.
Wk 6
Usability Testing and Evaluation
LectureCover moderated usability testing, the System Usability Scale, think-aloud protocols, heuristic evaluation, and how to turn findings into actionable redesign decisions.
PracticeConduct a moderated usability test of the week-5 prototype; score SUS and identify the top five usability problems.
ProjectFirst round of usability testing is complete; a redesign backlog prioritized by severity is produced.
Multimodal and Accessibility
Wk 7
Multimodal Interfaces: Voice and Gesture
LectureExamine multimodal fusion architectures, voice interface design (VUI) and conversational patterns, gesture recognition, and design trade-offs across input modalities.
PracticeAdd a voice command layer to a prototype using a speech recognition API and evaluate the interaction design.
ProjectA multimodal input channel (voice, gesture, or gaze) is designed and integrated into the prototype.
Wk 8
Interim Demo MilestonePresentation
LectureIntroduce gaze-based and haptic interaction models and mixed-reality HCI patterns for extended reality contexts.
PracticeTeams present an interactive Figma prototype with the multimodal layer and initial usability-test results.
ProjectHigh-fidelity prototype with multimodal interaction and first usability evidence is demonstrated.
Wk 9
Accessibility and Inclusive Design
LectureExamine WCAG 2.2 principles (perceivable, operable, understandable, robust), assistive technology compatibility, semantic HTML and ARIA, and inclusive design as a systematic practice.
PracticeAudit the project prototype with axe DevTools and WAVE; remediate at least three Level AA barriers and re-test.
ProjectProject prototype passes automated WCAG 2.2 Level AA audit and documents manual screen-reader testing.
AI-Augmented Interaction
Wk 10
Adaptive Interfaces and User Modeling
LectureIntroduce user modeling, context-aware personalization, recommendation in UI, and the design and evaluation of adaptive interfaces that respond to user behavior and preferences.
PracticeImplement a simple rule-based or ML-driven adaptive UI element in the project and evaluate its effect on task completion time.
ProjectProject includes a demonstrable adaptive or personalized interaction component.
Wk 11
Conversational UI and Intelligent Agents
LectureCover dialogue management design patterns, conversational UX heuristics, chatbot and voice assistant evaluation, and design guidelines for AI agent interaction.
PracticeDesign and prototype a conversational UI flow for the project using a large language model API; evaluate it with a structured user walkthrough.
ProjectProject includes a conversational or AI-agent interaction layer with defined dialogue flows and evaluation results.
Wk 12
A/B Testing and Data-Driven Design
LectureIntroduce controlled A/B and multivariate experiments, statistical power, sequential testing, and analytics instrumentation for behavior tracking in production interfaces.
PracticeDesign an A/B test for one design decision in the project; implement analytics events with a session-recording or analytics tool and analyze mock results.
ProjectProject includes an A/B test design with hypothesis, sample-size calculation, and an instrumented prototype.
Capstone
Wk 13
Final Demo and DefensePresentation
LectureSynthesize HMI theory, design methodology, and AI-augmented interaction; review evaluation evidence and design trade-offs.
PracticeTeams present the finished interface with an oral defense covering research, design decisions, accessibility audit, multimodal layer, AI-augmented features, and usability evidence.
ProjectFinal human-machine interface is delivered with design system, usability report, accessibility certification, and A/B test analysis.
AI tools in this course.

Students use AI assistants to generate wireframe descriptions, accessibility remediation suggestions, and usability test analysis summaries. They prompt tools to draft dialogue flows for conversational UI, suggest adaptive-interface rules from user-behavior data, and generate A/B test hypotheses from analytics patterns. AI helps create design system documentation and component specifications from Figma prototypes, and supports rapid iteration on prototype content. Because AI-generated UX copy and interaction flows can introduce inconsistency or violate established design principles, students validate every AI-assisted artifact against the cognitive models, usability heuristics, and accessibility standards covered in lecture before incorporating it into the project.

Student project

Teams design, prototype, and evaluate one human-machine interface for a real system or a well-defined domain problem. The interface evolves across 13 weeks from a user-research synthesis and low-fidelity concept sketch into a high-fidelity Figma prototype with a multimodal input channel, an AI-augmented layer, full WCAG 2.2 Level AA compliance, and a final usability evaluation report backed by two rounds of testing and an A/B test design.

Requirements

  • Build a working prototype, not a set of disconnected screens.
  • Be original: a new interface that solves a real user problem, not a reskin of an existing tutorial product.
  • Show real depth: real user research, real usability testing with at least five participants per round, and accessibility evidence from both automated and manual testing.
  • Carry one running project from user research through final evaluation across the whole term.
  • Work in a team of three or four and defend the design at each of the three presentations (weeks 5, 8, and 13).

Tools & platforms

  • Figma: design, prototype, and maintain the component library and design system
  • Miro: run ideation workshops, affinity diagrams, and journey mapping sessions
  • axe DevTools: audit the prototype for WCAG 2.2 Level AA compliance violations
  • WAVE: supplement automated accessibility evaluation with visual feedback overlays
  • System Usability Scale (SUS): quantify perceived usability across test sessions
  • Lookback: record and replay moderated usability test sessions remotely
  • Web Speech API: add voice recognition input to browser-based prototype implementations
  • PostHog: instrument interaction analytics and A/B testing in web prototypes
  • Python: analyze SUS scores, task-success rates, and A/B test results statistically
  • OpenAI API: implement conversational UI and intelligent agent layers in the prototype
