About this course

High-Performance Computing and Programming equips students with the theory and hands-on skills to exploit parallelism at every level of the hardware stack: from shared-memory threads and cache-coherent multiprocessors, through message-passing clusters, to massively parallel GPU architectures. The course opens with a structured survey of parallel and distributed programming paradigms: imperative thread-based models, functional and declarative models (where immutability eliminates entire classes of concurrency bugs), data-parallel models, actor-based models, and pipeline models. Students then study a comprehensive set of synchronization mechanisms, from classic mutexes and semaphores to monitors, barriers, condition variables, lock-free algorithms based on compare-and-swap, and software transactional memory. With that foundation in place, they learn to select the right model for a workload, implement correct and efficient parallel programs in OpenMP, MPI, and CUDA, and identify bottlenecks with profiling tools. The course concludes with a semester project in which teams build and benchmark a distributed computation pipeline on a real multi-node cluster.

Course format. Thirteen weeks, four contact hours each: a two-hour lecture (concepts and theory) and a two-hour practice session. The course is project-based; teams carry one running parallel application end to end and present it three times, in weeks 5, 9, and 13.
What you will build

Designed and implemented a parallelized computation pipeline that runs on a multi-node cluster: profiled a serial baseline with perf and VTune to locate hotspots, rewrote compute kernels with OpenMP for shared-memory speedup, added an MPI communication layer for inter-node distribution, ported the most compute-intensive kernel to CUDA for GPU acceleration, used NCCL for multi-GPU collective operations, managed job scheduling with Slurm, and produced a final benchmark report with roofline-model analysis and scalability curves for thread count, process count, and GPU count.

Expected outcomes

  • Compare parallel and distributed programming paradigms (imperative threads, functional/declarative, data-parallel, actor-based, pipeline) and select the appropriate model for a given workload
  • Explain parallel computer architectures: SMP, NUMA topology, cluster interconnect topologies, and GPU streaming multiprocessors
  • Apply synchronization mechanisms including mutexes, semaphores, monitors, barriers, lock-free algorithms (CAS, hazard pointers), and software transactional memory (STM)
  • Reason about functional concurrency: exploit immutability and referential transparency to eliminate data races, and implement parallel map and reduce patterns
  • Write correct OpenMP programs using work-sharing constructs, synchronization primitives, and task-based parallelism while avoiding data races and false sharing
  • Design and implement MPI programs using point-to-point and collective communication, handle process topology, and implement fault-tolerant patterns
  • Write CUDA kernels with appropriate grid and block configurations, manage device memory, and optimize for memory coalescing and occupancy
  • Apply the roofline model and Amdahl's Law to predict and interpret observed speedups
  • Profile serial and parallel programs with perf, VTune, and Nsight Compute to locate bottlenecks and guide optimization
  • Implement distributed fault tolerance using checkpoint-restart and redundancy strategies
  • Schedule and monitor parallel jobs on a Slurm-managed HPC cluster
  • Produce benchmark reports with scalability curves, roofline-model overlays, and efficiency analyses

Theoretical foundations

The concepts and results this course rests on.

  • Flynn's taxonomy of computer architectures: SISD, SIMD, MISD, MIMD
  • Parallel architecture classes: symmetric multiprocessing (SMP), non-uniform memory access (NUMA), distributed-memory clusters, interconnect topologies (fat-tree, torus, hypercube)
  • Cache coherence protocols: MSI, MESI, and MOESI; false sharing and cache-line padding
  • NUMA topology and its impact on memory access latency
  • Parallel programming paradigms: comparison of thread-based imperative, functional/declarative, data-parallel, actor-based, and pipeline models
  • Functional programming and concurrency: referential transparency, immutability as a parallelism safety guarantee, pure functions and side-effect isolation, parallel map and fold patterns
  • Synchronization primitives: mutexes, semaphores, monitors, condition variables, barriers, read-write locks
  • Lock-free concurrency: compare-and-swap (CAS), atomic fetch-and-add, the ABA problem, hazard pointers, memory ordering constraints
  • Software transactional memory (STM): optimistic concurrency, conflict detection, retry semantics, composability
  • Actor model: message-passing concurrency without shared state, mailboxes, supervision trees
  • Amdahl's Law and Gustafson's Law for speedup bounds
  • The roofline performance model: arithmetic intensity versus memory bandwidth
  • OpenMP memory model: flush, acquire, and release semantics
  • MPI communication semantics: blocking and non-blocking operations, collective algorithms
  • GPU architecture: streaming multiprocessors, warp execution, and the SIMT model
  • CUDA memory hierarchy: global, shared, constant, and texture memory
  • Memory coalescing and warp divergence as GPU performance limiters
  • CAP theorem and consistency models in distributed systems
  • Checkpoint-restart and message logging for distributed fault tolerance
  • Collective communication algorithms: reduce-scatter, all-gather, ring all-reduce
  • Work stealing and load balancing in task-parallel runtimes

Weekly schedule 13 weeks · lecture + practice

Shared-Memory Parallelism
Wk 1
Parallel Architectures and Programming Paradigms
LectureSurvey parallel and distributed architectures: SMP, NUMA, GPU clusters, and cluster interconnect topologies (fat-tree, torus). Introduce Flynn's taxonomy, Amdahl's Law, and Gustafson's Law. Survey the major programming paradigms: imperative thread-based, functional and declarative (immutability, pure functions), data-parallel (SIMD, map-reduce), actor-based (message-passing without shared state), and pipeline models. Discuss when each paradigm is the right tool.
PracticeBenchmark a serial program, measure clock cycles and cache misses with perf, identify the dominant bottleneck, and classify it by paradigm suitability.
ProjectProject domain is chosen; serial baseline application is selected and compiled with profiling instrumentation.
Wk 2
Synchronization Mechanisms and Functional Concurrency
LectureSurvey the full synchronization toolkit: mutexes, semaphores, monitors, condition variables, read-write locks, and barriers. Introduce lock-free concurrency: compare-and-swap (CAS), atomic fetch-and-add, the ABA problem, and hazard pointers. Introduce software transactional memory (STM): optimistic concurrency, conflict detection, and retry semantics. Contrast with the functional approach: how immutability and referential transparency eliminate shared-state bugs entirely; parallel map, fold, and reduce as side-effect-free patterns. Cover the C++ memory model, happens-before relations, and common bugs: races, deadlocks, and livelocks.
PracticeImplement a producer-consumer pipeline with pthreads; reproduce a data race with ThreadSanitizer and fix it first with a mutex, then with a lock-free queue using CAS, and finally with a pure-functional immutable variant.
ProjectSerial hotspot is identified and instrumented; the team selects a synchronization strategy (lock-based, lock-free, or functional) and documents the rationale.
Wk 3
OpenMP: Work-Sharing and Synchronization
LectureIntroduce OpenMP pragmas: parallel regions, for work-sharing, sections, reduction, barrier, critical, and atomic. Discuss false sharing and cache-line padding.
PracticeParallelize a matrix multiplication kernel with OpenMP and measure speedup versus thread count.
ProjectFirst OpenMP parallelization of the project's dominant loop is implemented and benchmarked.
Wk 4
OpenMP Advanced: Tasks, SIMD, and Offloading
LectureCover OpenMP task parallelism, depend clauses, target offloading, and SIMD vectorization directives. Introduce the roofline model for performance ceilings.
PracticeImplement a task-parallel divide-and-conquer algorithm (mergesort or quicksort) with OpenMP tasks and compare with the work-sharing version.
ProjectRoofline model is constructed for the project baseline; OpenMP task parallelism is applied where recursion is present.
Message-Passing Parallelism
Wk 5
MPI Fundamentals and Checkpoint 1Presentation
LectureIntroduce MPI: communicators, point-to-point operations (send, recv, isend, irecv), and the MPI execution model. Compare shared-memory and distributed-memory trade-offs.
PracticeTeams present their serial baseline, profiling results, OpenMP implementation, and speedup curves. Q&A on parallelization strategy.
ProjectCheckpoint 1 is delivered: profiled baseline, OpenMP implementation, and scalability analysis up to 16 threads.
Wk 6
MPI Collective Communication
LectureCover MPI collectives: broadcast, scatter, gather, reduce, allreduce, and alltoall. Explain the ring all-reduce algorithm and its role in distributed deep learning.
PracticeImplement a distributed histogram using MPI scatter, local reduce, and MPI allreduce; measure latency versus bandwidth trade-offs.
ProjectMPI communication layer is added to the project; inter-process data distribution and aggregation are implemented.
Wk 7
Distributed Fault Tolerance and Consistency
LectureIntroduce the CAP theorem, consistency models, checkpoint-restart protocols, and message-logging fault-tolerance strategies for long-running HPC jobs.
PracticeAdd checkpoint-restart capability to the MPI application; simulate a process failure and verify recovery from the last checkpoint.
ProjectProject MPI implementation includes checkpoint-restart; a fault injection test is run and documented.
GPU Programming
Wk 8
GPU Architecture and CUDA Basics
LectureDescribe GPU hardware: streaming multiprocessors, CUDA cores, warp execution, the SIMT model, and the CUDA programming model: grids, blocks, and threads.
PracticeWrite a CUDA kernel for vector addition and matrix multiplication; compare performance with the CPU baseline.
ProjectA CUDA prototype for the project's most compute-intensive kernel is implemented and compiles correctly.
Wk 9
CUDA Memory Optimization and Checkpoint 2Presentation
LectureCover shared memory, memory coalescing, bank conflicts, constant memory, and warp divergence. Introduce Nsight Compute for GPU profiling and occupancy analysis.
PracticeTeams present MPI implementation, checkpoint-restart design, and scalability on a multi-node cluster. Q&A on distribution strategy.
ProjectCheckpoint 2 is delivered: MPI multi-node implementation with fault tolerance and scalability curves across 1, 2, 4, and 8 processes.
Wk 10
Multi-GPU Programming and NCCL
LectureIntroduce multi-GPU topologies, peer-to-peer memory access, and the NCCL library for GPU-optimized collective operations. Discuss hybrid MPI+CUDA architectures.
PracticeExtend the CUDA kernel to use two GPUs with NCCL all-reduce; measure bandwidth and compare with a single-GPU baseline.
ProjectCUDA kernel is memory-optimized using shared memory and profiled with Nsight Compute; occupancy is reported.
Performance Engineering and Scheduling
Wk 11
Performance Profiling and Roofline Analysis
LectureIntegrate profiling tools: perf, Intel VTune, and NVIDIA Nsight Compute. Construct roofline models for CPU and GPU kernels, identify memory-bound versus compute-bound regions.
PracticeProfile the full project pipeline end to end; overlay measured kernel performance on the roofline model and identify the next optimization target.
ProjectFull-pipeline profile is produced; roofline-model chart is generated and the dominant bottleneck for the final optimization pass is identified.
Wk 12
HPC Job Scheduling with Slurm
LectureIntroduce Slurm workload management: job scripts, resource allocation (nodes, tasks, GPUs), array jobs, and monitoring commands. Discuss fair-share scheduling and queue policies.
PracticeWrite Slurm job scripts for the project pipeline; submit strong-scaling and weak-scaling sweeps and parse the output into scalability curves.
ProjectFull pipeline runs on the cluster via Slurm; strong-scaling and weak-scaling experiments across 1 to 8 nodes are submitted and results are collected.
Capstone
Wk 13
Final Demo and DefensePresentation
LectureSynthesize the course: parallel algorithm design, memory hierarchy exploitation, GPU acceleration, and distributed fault tolerance. Review final benchmark results and design trade-offs.
PracticeTeams present the finished parallelized pipeline with an oral defense covering profiling findings, OpenMP and MPI design decisions, CUDA kernel optimization, and scalability evidence.

AI tools in this course.

Students use AI assistants to generate parallel algorithm skeletons, suggest CUDA kernel grid configurations, and draft Slurm job scripts from workload descriptions. They prompt tools to explain profiler output, interpret roofline charts, and propose memory-layout optimizations for coalescing. AI helps generate MPI communication patterns for standard decomposition strategies and can draft checkpoint-restart scaffolding. Because AI-generated parallel code can contain subtle races, incorrect synchronization, or wrong memory qualifiers, students must verify every AI-assisted implementation with ThreadSanitizer, CUDA-MEMCHECK, or MPI correctness tools before benchmarking and must understand the theoretical basis of every construct before including it in the final deliverable.

Student project

Teams select a compute-intensive application, produce a profiled serial baseline, and progressively parallelize it across shared memory, distributed memory, and GPU hardware over 13 weeks. The final deliverable is a Slurm-managed multi-node pipeline that integrates OpenMP, MPI, and CUDA, backed by a benchmark report with roofline-model analysis and scalability curves for thread count, process count, and GPU count.

Requirements

  • Start from a real, non-trivial serial application with a measurable compute bottleneck.
  • Implement at least two parallel levels: shared memory (OpenMP) and distributed memory (MPI) or GPU (CUDA).
  • Profile every version with a hardware-counter tool and document the roofline position of each kernel.
  • Demonstrate correct results: parallel outputs must match serial outputs for all test inputs.
  • Run strong-scaling and weak-scaling experiments and report parallel efficiency at each scale.
  • Work in a team of three or four and defend the implementation at each of the three presentations (weeks 5, 9, and 13).

Tools & platforms

  • OpenMP: shared-memory parallelism via compiler directives for C, C++, and Fortran
  • MPI (Open MPI): message-passing library for distributed-memory multi-node parallel programs
  • CUDA Toolkit: NVIDIA GPU programming model, compiler (nvcc), and runtime libraries
  • NCCL: NVIDIA Collective Communications Library for multi-GPU all-reduce and broadcast
  • perf: Linux hardware-counter profiler for CPU cycle, cache miss, and branch analysis
  • Intel VTune Profiler: guided performance analysis, hotspot detection, and threading efficiency
  • NVIDIA Nsight Compute: CUDA kernel profiler with occupancy, warp efficiency, and memory throughput metrics
  • Slurm: workload manager for submitting and monitoring parallel jobs on HPC clusters
  • ThreadSanitizer: dynamic data-race detector for multithreaded C and C++ programs
  • CUDA-MEMCHECK: memory error and race detection tool for CUDA programs
