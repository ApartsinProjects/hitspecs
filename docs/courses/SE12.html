About this course

Modern Storage and Information Retrieval Systems covers the algorithms and architectures that underpin search engines, compressed databases, and hybrid retrieval pipelines. Students build inverted indexes from raw text corpora, implement TF-IDF and BM25 ranking functions, study the mathematics of lossless compression (Huffman, LZ-family, arithmetic coding), construct approximate nearest-neighbour (ANN) indexes with HNSW, and combine lexical and dense vector retrieval into a unified hybrid pipeline. Every concept is grounded in a semester-long project that ships a working compressed and queryable corpus.

Course format. Thirteen weeks, four contact hours each: a two-hour lecture (theory and algorithms) and a two-hour practice session (implementation and profiling). The course is project-based; teams carry one running project end to end and present it three times, in weeks 5, 8, and 13.
What you will build

Designed and implemented a compressed full-text search engine: tokenized and indexed a multi-gigabyte text corpus into an inverted index with variable-byte and PForDelta posting compression, implemented TF-IDF and BM25 ranking with efficient DAAT traversal, encoded the static corpus with Huffman and Brotli to reduce storage footprint, added a dense retrieval layer using SentenceTransformers and a FAISS HNSW index for semantic search, and fused lexical and dense scores into a hybrid pipeline exposed through a REST API with sub-100 ms latency on a commodity machine.

Expected outcomes

  • Construct and query an inverted index from a large text corpus, including tokenization, stemming, stop-word removal, and posting list compression
  • Implement and compare TF-IDF and BM25 ranking functions, analyzing their sensitivity to term frequency saturation and document length normalization
  • Apply Huffman, LZ77/LZ78, and arithmetic coding to compress structured and unstructured data, and measure compression ratio and decode throughput
  • Build and query a FAISS HNSW approximate nearest-neighbour index, tuning construction and search parameters to balance recall and latency
  • Generate and evaluate dense vector embeddings with SentenceTransformers, understanding the role of contrastive training and cosine similarity in semantic ranking
  • Design and implement a hybrid retrieval pipeline that fuses BM25 lexical scores with dense retrieval scores using reciprocal rank fusion or learned interpolation
  • Profile index construction and query throughput with real benchmarking tools and produce a latency-recall Pareto analysis
  • Deploy a compressed, queryable corpus as a REST service with documented API, benchmark results, and a final engineering report

Theoretical foundations

The concepts and results this course rests on.

  • Information theory: Shannon entropy, source coding theorem, and limits of lossless compression
  • Inverted index model: document frequency, term frequency, postings lists, and skip pointers
  • Vector space model and cosine similarity for document ranking
  • TF-IDF weighting: term frequency saturation and inverse document frequency
  • Okapi BM25: probabilistic relevance model, k1 and b parameters, document-length normalization
  • Huffman coding: optimal prefix-free codes, greedy construction, and canonical Huffman
  • LZ77 and LZ78: sliding-window and dictionary-based compression, DEFLATE and zlib
  • Arithmetic coding: interval encoding, adaptive models, and range coding
  • Variable-byte and PForDelta integer compression for posting lists
  • DAAT (Document-At-A-Time) and TAAT (Term-At-A-Time) query evaluation strategies
  • Locality-sensitive hashing (LSH) and product quantization for approximate search
  • Hierarchical Navigable Small World (HNSW) graph structure and greedy layer search
  • Contrastive learning and bi-encoder architecture for dense retrieval
  • Reciprocal rank fusion (RRF) and linear interpolation for hybrid ranking

Weekly schedule 13 weeks · lecture + practice

Indexing and Retrieval
Wk 1
Inverted Index Construction
LectureIntroduce the inverted index model: document collection, tokenization, stemming, stop-word removal, term dictionaries, and posting lists with document and term frequencies.
PracticeBuild a simple in-memory inverted index over a 100 K document subset of Wikipedia; measure index size and lookup latency.
ProjectProject domain and target corpus are selected; the team downloads and characterizes the dataset.
Wk 2
Vector Space Model and TF-IDF Ranking
LectureCover the vector space model, cosine similarity, TF-IDF weighting variants (raw, log, augmented), and the impact of IDF on rare vs. common terms.
PracticeExtend the week-1 index to support TF-IDF scoring; rank results for a test query set and evaluate with precision@10.
ProjectCorpus preprocessing pipeline (tokenization, normalization, deduplication) is finalized and documented.
Wk 3
BM25 and Probabilistic Ranking
LectureDerive BM25 from the probabilistic relevance model; analyze the k1 and b parameters, document-length normalization, and performance on standard TREC benchmarks.
PracticeImplement BM25 on the existing index; compare BM25 vs. TF-IDF ranking quality on a labeled relevance set using NDCG and MAP.
ProjectBM25 baseline search over the project corpus is operational with basic query evaluation metrics.
Wk 4
Query Processing and Posting List Compression
LecturePresent DAAT and TAAT query evaluation, skip pointers, AND/OR/NOT Boolean query processing, and integer compression schemes: variable-byte, gamma, delta, and PForDelta encoding.
PracticeAdd posting list compression to the index using variable-byte encoding; measure the reduction in index size and query latency change.
ProjectProject index supports compressed postings and Boolean query operators with correct result sets.
Compression Algorithms
Wk 5
Information Theory and Huffman CodingPresentation
LectureIntroduce Shannon entropy, the source coding theorem, and optimal prefix-free codes. Derive Huffman coding, canonical Huffman trees, and adaptive Huffman for streaming data.
PracticeTeams present the project specification: corpus characterization, index architecture, BM25 baseline evaluation, and a plan for compression and vector retrieval extensions.
ProjectApproved specification with index design, BM25 baseline NDCG scores, and project roadmap is delivered.
Wk 6
LZ-Family and Dictionary Compression
LectureCover LZ77 sliding-window compression, LZ78 dictionary trie, LZSS, LZW, and how DEFLATE combines LZ77 with Huffman. Compare compression ratios and throughputs across zlib, LZ4, Zstandard, and Brotli.
PracticeBenchmark zlib, LZ4, and Zstandard on the project corpus and on posting-list byte arrays; plot the compression ratio vs. decode throughput Pareto frontier.
ProjectCorpus is stored in at least two compressed formats; storage footprint and decode latency are measured and reported.
Wk 7
Arithmetic Coding and Context Modeling
LectureIntroduce arithmetic coding as a practical implementation of Shannon's bound, range coding, context-mixing models, and the PAQ family. Compare with Huffman on natural language text.
PracticeImplement a simple order-0 arithmetic coder and compare its ratio with Huffman on the project corpus; profile encoding and decoding speed.
ProjectProject compression module is complete with benchmarked ratio, encode/decode throughput, and a written analysis of the entropy of the corpus.
Vector and Approximate Search
Wk 8
Dense Embeddings and Semantic RetrievalPresentation
LectureIntroduce dense retrieval: bi-encoder architecture, contrastive training (SimCSE, MS-MARCO), cosine similarity vs. dot product, and the semantic vs. lexical retrieval gap.
PracticeTeams demonstrate the project index with compression benchmarks and BM25 ranking results; present the design for the dense retrieval extension.
ProjectCompressed index with BM25 baseline and interim benchmark results is presented; dense retrieval extension design is approved.
Wk 9
Approximate Nearest-Neighbour Search and HNSW
LectureCover the ANN problem, LSH, product quantization, and HNSW: layer construction, greedy graph traversal, ef_construction vs. ef_search parameters, and recall-latency tradeoffs.
PracticeBuild an HNSW index over SentenceTransformer embeddings of the project corpus using FAISS; measure recall@10 and query latency at varying ef_search.
ProjectDense index (HNSW) is built over the project corpus; recall and latency are measured and plotted against ef_search.
Wk 10
FAISS Index Types and Quantization
LectureSurvey FAISS index families: Flat, IVF, PQ, IVFPQ, HNSW, and IVFHNSW. Explain product quantization and scalar quantization as memory reduction techniques. Discuss GPU acceleration of FAISS indexes.
PracticeCompare Flat, IVF, and IVFPQ indexes on the project embeddings for recall, latency, and index size; select the best index for the project's latency target.
ProjectFinal FAISS index type is selected and justified with a recall-latency-memory benchmark table.
Hybrid Search and Production
Wk 11
Hybrid Lexical and Dense Retrieval
LectureIntroduce hybrid retrieval: reciprocal rank fusion (RRF), linear score interpolation, late interaction (ColBERT), and learned sparse retrieval (SPLADE). Analyze when each approach outperforms pure BM25 or pure dense retrieval.
PracticeImplement RRF over BM25 and HNSW result lists on the project corpus; measure NDCG and MRR improvements over each single-retriever baseline.
ProjectHybrid retrieval pipeline is operational; evaluation metrics for BM25-only, dense-only, and hybrid are tabulated.
Wk 12
Production Indexing and Storage at Scale
LectureCover Elasticsearch and Apache Lucene architecture: segment merging, commit points, refresh cycles, inverted file format on disk, and operational considerations for large-scale search.
PracticeIngest the project corpus into Elasticsearch; compare its BM25 and hybrid kNN performance against the hand-built pipeline for accuracy and throughput.
ProjectProject REST API is complete: accepts a text query, calls BM25 and dense retrievers, applies RRF, and returns ranked results with latency under 100 ms locally.
Capstone
Wk 13
Final Demo and DefensePresentation
LectureSynthesize the course: index architectures, compression theory, approximate search, and hybrid retrieval; review the engineering tradeoffs and open research directions.
PracticeTeams present the finished hybrid search system with a live demo, benchmark report covering compression ratios, latency-recall curves, and NDCG comparisons across retrieval modes.
ProjectFinal compressed full-text search system is delivered: source code, REST API, benchmark report, and a written engineering analysis.
AI tools in this course.

Students use AI assistants to generate tokenization and normalization pipelines, suggest compression parameter tuning strategies, and draft benchmark analysis narratives. They prompt tools to explain HNSW construction edge cases, generate SentenceTransformer fine-tuning recipes, and produce boilerplate REST API scaffolding. Because AI-generated compression and indexing code can introduce subtle correctness errors (wrong variable-byte decoding, off-by-one in postings), students validate every AI-assisted implementation against known-good test vectors and documented algorithm specifications before integrating it into the project.

Student project

Teams build a compressed, searchable corpus over a multi-gigabyte real-world text collection. The system evolves from a raw inverted index in week 1 into a full hybrid retrieval pipeline in week 13, adding BM25 ranking, posting-list compression, corpus-level lossless compression, a dense FAISS HNSW index over SentenceTransformer embeddings, and a REST API with hybrid BM25-dense ranking. The final deliverable includes source code, a live demo, latency-recall benchmarks, and an engineering report analyzing compression tradeoffs and retrieval quality.

Requirements

  • Build a working search pipeline, not a toy: index at least 100 K real documents.
  • Implement BM25 and at least one compression algorithm from scratch (not just call a library).
  • Include a dense retrieval path with a real embedding model and a FAISS index.
  • Measure and report NDCG, MRR, compression ratio, and query latency for every retrieval mode.
  • Work in a team of three or four and defend the system at each of the three presentations (weeks 5, 8, and 13).

Tools & platforms

  • Elasticsearch: production-grade full-text search with BM25 ranking and native kNN dense retrieval
  • Apache Lucene: reference inverted index implementation; study segment structure, codec internals, and query parsing
  • FAISS: efficient similarity search library supporting Flat, IVF, PQ, and HNSW index types
  • SentenceTransformers: pre-trained bi-encoder models for generating dense text embeddings
  • zlib / Brotli: reference implementations of DEFLATE and Brotli for corpus and posting list compression
  • LZ4 / Zstandard: high-throughput compression alternatives for latency-sensitive index storage
  • Python: primary implementation language for index construction, ranking, and benchmarking scripts
  • NumPy: vectorized computation for embedding similarity and evaluation metric calculation
  • FastAPI: REST API framework for the project's query endpoint and benchmark harness
